# GitHub Pages 部署指南

## 第一步：在GitHub创建仓库

1. 访问 https://github.com 并登录
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: xinghui-shixi
   - 选择 Public（必须，否则GitHub Pages需要付费）
   - 不要勾选 "Add a README file"
4. 点击 "Create repository"
5. 复制仓库URL（类似 https://github.com/你的用户名/xinghui-shixi.git）

## 第二步：推送代码到GitHub

打开终端，执行以下命令：

```bash
cd /Users/niuniu/WorkBuddy/20260423202339

# 添加远程仓库（替换成你的仓库URL）
git remote add origin https://github.com/你的用户名/仓库名.git

# 设置默认分支为main
git branch -M main

# 推送到GitHub
git push -u origin main
```

输入GitHub用户名和密码（或Personal Access Token）

## 第三步：启用GitHub Pages

1. 在GitHub仓库页面，点击 "Settings"
2. 左侧菜单找到 "Pages"
3. Source 选择 "main" 分支
4. 点击 "Save"
5. 等待2-3分钟，页面会显示：
   "Your site is live at https://你的用户名.github.io/仓库名/"

## 第四步：访问网站

- PC端：https://你的用户名.github.io/仓库名/
- 移动端：https://你的用户名.github.io/仓库名/mobile.html

## 常见问题

### Q: 推送时提示需要输入用户名和密码？
A: GitHub已不支持密码验证，需要使用Personal Access Token：
   1. 访问 GitHub → Settings → Developer settings → Personal access tokens
   2. 生成新token（勾选repo权限）
   3. 使用token作为密码

### Q: 网站访问显示404？
A: 检查是否已将分支设置为main，且文件已成功推送

### Q: 中文显示乱码？
A: 确保index.html文件头部有 <meta charset="UTF-8">

### Q: 如何更新网站内容？
A: 修改文件后，执行：
   git add .
   git commit -m "更新说明"
   git push
   等待1-2分钟，GitHub Pages会自动更新
