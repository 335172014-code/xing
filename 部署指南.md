# 部署指南 - 实习智能匹配平台 v2.0

## 架构概览

```
用户浏览器 ──→ Vercel (HTTPS)
                  ├── /public/*   → 静态文件 (index.html, admin.html)
                  └── /api/*      → Serverless Functions (auth, search, stats, tokens)
                                      └── Supabase PostgreSQL (数据库)
```

## 一、Supabase 配置

### 1. 创建 Supabase 项目

1. 访问 [https://supabase.com](https://supabase.com) 注册并登录
2. 点击 "New Project"，选择组织，填写项目名称和数据库密码
3. 选择离你最近的区域（推荐：Northeast Asia - Tokyo）
4. 等待项目创建完成（约2分钟）

### 2. 执行数据库建表SQL

1. 进入项目后，点击左侧 "SQL Editor"
2. 复制 `supabase/schema.sql` 的全部内容，粘贴到编辑器
3. 点击 "Run" 执行

### 3. 获取连接信息

1. 点击左侧 "Settings" → "API"
2. 记录以下信息：
   - **Project URL**: `https://xxxx.supabase.co`
   - **service_role key**: `eyJ...`（注意：这是服务端密钥，不要泄露！）

## 二、Vercel 配置

### 1. 安装 Vercel CLI

```bash
npm install -g vercel
```

### 2. 本地开发

```bash
# 安装依赖
npm install

# 创建本地环境变量文件
cp .env.example .env.local

# 编辑 .env.local，填入实际值
# SUPABASE_URL=https://xxxx.supabase.co
# SUPABASE_SERVICE_KEY=eyJ...
# ADMIN_KEY=your-secure-admin-password

# 启动本地开发服务器
npm run dev
```

### 3. 部署到 Vercel

```bash
# 登录 Vercel
vercel login

# 部署（首次会引导创建项目）
vercel

# 生产环境部署
vercel --prod
```

### 4. 配置环境变量

在 Vercel Dashboard 中配置环境变量：

1. 进入项目 Settings → Environment Variables
2. 添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `SUPABASE_URL` | `https://xxxx.supabase.co` | Supabase项目URL |
| `SUPABASE_SERVICE_KEY` | `eyJ...` | Supabase服务端密钥 |
| `ADMIN_KEY` | 自定义强密码 | 管理看板访问密码 |

## 三、创建第一个Token

部署完成后，访问管理看板创建Token：

1. 访问 `https://your-domain.vercel.app/admin.html`
2. 输入 `ADMIN_KEY` 登录
3. 在"创建新Token"区域输入机构名称，点击"生成Token"
4. 将生成的Token发送给使用者

## 四、使用流程

1. 使用者访问 `https://your-domain.vercel.app`
2. 输入管理员分配的Token
3. 验证通过后即可搜索岗位
4. 搜索次数会在Token配额中扣除

## 五、环境变量文件

创建 `.env.local`（本地开发用，不要提交到Git）：

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=eyJyour-service-key
ADMIN_KEY=your-secure-admin-password
```

创建 `.env.example`（可提交到Git的示例文件）：

```env
SUPABASE_URL=
SUPABASE_SERVICE_KEY=
ADMIN_KEY=
```

## 六、安全注意事项

1. **永远不要**将 `.env.local` 提交到 Git
2. **永远不要**在前端代码中硬编码任何密钥
3. `SUPABASE_SERVICE_KEY` 拥有完全数据库访问权限，必须保密
4. `ADMIN_KEY` 应该使用强密码（建议16位以上，包含大小写字母和数字）
5. 定期检查Token使用情况，禁用异常Token
6. 建议为每个机构/使用者创建独立Token

## 九、故障排查

### 搜索返回401
- Token已过期或被禁用，联系管理员

### 搜索返回403
- 搜索次数已用完，管理员可增加配额

### 搜索返回429
- 搜索频率过高（每分钟30次限制），稍后重试

### 搜索返回500
- 检查Supabase连接是否正常
- 检查环境变量是否正确配置
- 查看Vercel Functions日志

### 管理看板无法加载
- 检查ADMIN_KEY是否正确
- 检查Supabase连接

### 岗位数据为空
- 确认 `data/jobs_data.json` 文件存在于部署中
- 检查文件是否完整（约2MB）
